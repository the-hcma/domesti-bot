# Publish `domesti-bot` on PyPI (pipx / `pip install`)

## Goal

Publish `domesti-bot` to [PyPI](https://pypi.org/) so users can install and run:

- **`pipx install domesti-bot`** (recommended) or `pip install domesti-bot`
- **`domesti-bot`** ‚Äî interactive REPL CLI (`app.domesti_bot_cli:main`)
- **`domesti-bot-server`** ‚Äî HTTP API + web UI (`config.serve:main`)

Releases are **automated on `main`**: Conventional Commits ‚Üí [Release Please](https://github.com/googleapis/release-please) opens/updates a release PR ‚Üí merging that PR creates a Git tag and triggers CI to build and **`uv publish`** to PyPI, followed by a post-publish smoke test.

---

## Version and build identity

One resolver ‚Äî **`app.build_info.get_build_info()`** ‚Üí `(version, commit)` ‚Äî must feed every user-visible surface. PyPI wheels and shallow containers have no `.git` checkout, so the **publish pipeline embeds** version and commit into the wheel at build time.

### Canonical version (releases)

| Layer | Role |
| --- | --- |
| **`pyproject.toml` ‚Üí `[project].version`** | Single source of truth for the package (today `0.1.0`). Release Please bumps this field and `CHANGELOG.md` on the release PR. |
| **Git tag** | Created when the release PR merges (e.g. `v0.2.0`). Tag name should match `[project].version`. |
| **`scripts/embed_build_metadata.py`** | Run **once per release build** in CI, **before** `uv build`, with `DOMESTI_EMBED_VERSION` and `DOMESTI_EMBED_COMMIT` set from the release. Writes `app/_build_metadata.py` (`EMBEDDED_VERSION`, `EMBEDDED_COMMIT`) into the artifact that lands in the wheel. |
| **`uv build` / `uv publish`** | Produces a wheel/sdist whose installed metadata also carries `version` via standard Python packaging (`importlib.metadata.version("domesti-bot")`). |

**Publish CI must set:**

```bash
export DOMESTI_EMBED_VERSION="$(python -c 'import tomllib; print(tomllib.load(open("pyproject.toml","rb"))["project"]["version"])')"
export DOMESTI_EMBED_COMMIT="${GITHUB_SHA}"   # or short SHA from the release tag
uv run python scripts/embed_build_metadata.py
uv build
uv publish
```

`app/_build_metadata.py` is autogenerated ‚Äî do not hand-edit; it may be gitignored or committed only in release artifacts depending on implementation choice (prefer **generate in CI only** and include the file inside the built wheel via hatchling).

### Resolution at runtime (`get_build_info`)

Implemented in `app/build_info.py` (see tests in `tests/python/test_build_info.py`):

**Version** (first match wins):

1. `app._build_metadata.EMBEDDED_VERSION` (non-empty) ‚Äî PyPI wheel / release CI build
2. `importlib.metadata.version("domesti-bot")` ‚Äî normal `pip` / `pipx` install
3. `[project].version` in `pyproject.toml` ‚Äî editable / source checkout
4. Fallback `"0.0.0"` if unreadable

**Commit** (first match wins):

1. `GITHUB_SHA` or `DOMESTI_GIT_COMMIT` env (12-char hex) ‚Äî CI / deploy
2. `app._build_metadata.EMBEDDED_COMMIT` (non-empty)
3. `git rev-parse --short=12 HEAD` when `.git` exists
4. `"unknown"`

### Where version and commit must appear

| Surface | Today | Target for PyPI |
| --- | --- | --- |
| **Web UI ‚Äî About dialog** | `GET /v1/meta` ‚Üí `MetaOut.version` / `.commit` via `appendAboutContent()` | Same; server must return embedded values after install |
| **Web UI ‚Äî bootstrap** | `DomestiBotController` calls `api.fetchMeta()` on init | Unchanged |
| **HTTP ‚Äî OpenAPI** | FastAPI `version=get_build_info()[0]` in `create_app()` | Same string as package version |
| **`GET /v1/meta`** | Public route (no API key); returns `MetaOut` | Contract tests in `tests/python/test_api_meta.py` |
| **CLI ‚Äî `domesti-bot`** | No `--version` flag yet | Add `--version` printing `version` and `commit` from `get_build_info()` |
| **CLI ‚Äî `domesti-bot-server`** | No `--version` flag yet | Add `--version` (same format) before uvicorn starts |
| **REPL banner** (optional) | May not show build info today | Consider one line on startup using `get_build_info()` so pipx users see version without opening the web UI |

**Format for CLI `--version`:** one line, easy to script, e.g. `domesti-bot 0.2.0 (abc123def456)` ‚Äî match whatever smoke tests assert.

**Rule:** never duplicate version strings in TypeScript or shell wrappers; the web client always reads the server via `/v1/meta`, and CLIs call `get_build_info()` in Python.
